# splitmerge-tool

[English](README.en.md)

基于浏览器的文件拆分/合并工具，支持把大文件拆分为 `.pkg` 分片，并自动生成可一键检测完整性与合并下载的 HTML 合并器页面。

## 功能特性

- 拆分输出为 `.pkg` 格式分片
- 拆分大小支持 `KB` / `MB`
- 纯 `HTML + JavaScript` 本地运行，无需后端
- 拆分后自动生成独立合并器（`.merge.html`）
- 一键检测分片完整性：
  - 是否缺少分片
  - 缺少哪些分片
  - 哈希校验是否一致
- 支持重命名合并后的输出文件名
- 支持随机分片前缀、每片随机文件名
- 下载模式支持：
  - 批量下载（单个 ZIP）
  - 单个逐一下载（减少弹窗骚扰）
- 界面默认中文，支持中英文一键切换
- 生成的合并器页面同样支持中英文切换

## 项目结构

- `index.html`: 主工具页面（拆分 + 生成合并器）
- `splitmerge_desktop.py`: 桌面版工具（不依赖浏览器）
- `build_exe.bat`: Windows 一键打包脚本（生成 exe）

## 桌面版（EXE）使用

如果你不想依赖浏览器，可以使用桌面版：

1. 安装 Python 3.10+（并勾选加入 PATH）。
2. 在项目目录运行：`build_exe.bat`
3. 打包完成后，在 `dist/splitmerge-desktop.exe` 启动程序。

桌面版能力：

- 界面默认中文，支持一键切换英文
- 拆分页签：
  - 拆分输出为 `.pkg`
  - 自动在源文件同目录生成输出文件夹：`<前缀>_output_时间戳`
  - 输出 `*.manifest.pkg.json`
  - 可自动将主程序（`exe` 或 `py`）复制到输出文件夹，便于直接带走使用
- 合并页签：
  - 读取 `manifest`
  - 校验缺失分片与哈希
  - 通过后合并为原文件（或指定输出名）

## 快速开始

1. 克隆或下载本仓库。
2. 直接用浏览器打开 `index.html`。
3. 选择要拆分的源文件。
4. 设置拆分大小（KB/MB）、输出文件名、分片命名方式。
5. 点击“开始拆分并生成”。
6. 选择下载模式：
   - 批量 ZIP 下载
   - 单个下载列表

## 生成文件说明

拆分后会生成以下文件：

- `*.part0001.pkg`, `*.part0002.pkg`, ...（或随机名 `.pkg`）
- `*.manifest.pkg.json`：分片清单与哈希信息
- `*.merge.html`：独立合并器页面

## 分片命名选项说明

- 使用随机分片前缀：
  - 会把“分片前缀”替换成随机字符串（如 `a8k2m1x9qz`）。
  - 顺序命名模式下，分片会变成 `a8k2m1x9qz.part0001.pkg` 这类形式。
  - 同时会影响自动生成文件名，例如 `a8k2m1x9qz.manifest.pkg.json` 与 `a8k2m1x9qz.merge.html`。
- 每个分片都使用随机文件名：
  - 每个 `.pkg` 分片会独立随机命名（如 `k3f8p2m1t0ab.pkg`、`q1z7c9n4r2de.pkg`）。
  - 不再使用 `前缀.part0001.pkg` 这种连续编号形式。
  - 合并时无需手动排序，生成的 `manifest` 会记录映射关系，合并器按清单自动校验并按顺序拼接。

## 合并流程

1. 打开生成的 `*.merge.html`。
2. 选择全部 `.pkg` 分片文件。
3. 点击“检测完整性”。
4. 若检测通过，点击“合并并下载”恢复原文件（或重命名后的目标文件）。

## 浏览器兼容性

推荐使用最新版 Chrome / Edge / Firefox（需支持）：

- `File API`
- `Blob`
- `Web Crypto API`（SHA-256）

## 注意事项

- 批量 ZIP 导出依赖 CDN 的 `JSZip`。当网络不可用或加载失败时，会自动回退到“单个下载列表”模式。
- 如果浏览器拦截下载，请允许当前站点下载或使用“单个逐一下载”。
- 超大文件处理时，受浏览器内存限制影响，建议适当减小分片大小。
