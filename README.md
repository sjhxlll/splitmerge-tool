# splitmerge-tool

[English](README.en.md)

基于浏览器的文件拆分/合并工具，支持把大文件拆分为 `.pkg` 分片，并自动生成可一键检测完整性与合并下载的 HTML 合并器页面。

## 功能特性

- 拆分输出为 `.pkg` 格式分片
- 拆分大小支持 `KB` / `MB`
- 纯 `HTML + JavaScript` 本地运行，无需后端
- 拆分后自动生成独立合并器（`.merge.html`）
- 一键检测分片完整性：
  - 是否缺少分片
  - 缺少哪些分片
  - 哈希校验是否一致
- 支持重命名合并后的输出文件名
- 支持随机分片前缀、每片随机文件名
- 下载模式支持：
  - 批量下载（单个 ZIP）
  - 单个逐一下载（减少弹窗骚扰）
- 界面默认中文，支持中英文一键切换
- 生成的合并器页面同样支持中英文切换

## 项目结构

- `index.html`: 主工具页面（拆分 + 生成合并器）

## 快速开始

1. 克隆或下载本仓库。
2. 直接用浏览器打开 `index.html`。
3. 选择要拆分的源文件。
4. 设置拆分大小（KB/MB）、输出文件名、分片命名方式。
5. 点击“开始拆分并生成”。
6. 选择下载模式：
   - 批量 ZIP 下载
   - 单个下载列表

## 生成文件说明

拆分后会生成以下文件：

- `*.part0001.pkg`, `*.part0002.pkg`, ...（或随机名 `.pkg`）
- `*.manifest.pkg.json`：分片清单与哈希信息
- `*.merge.html`：独立合并器页面

## 合并流程

1. 打开生成的 `*.merge.html`。
2. 选择全部 `.pkg` 分片文件。
3. 点击“检测完整性”。
4. 若检测通过，点击“合并并下载”恢复原文件（或重命名后的目标文件）。

## 浏览器兼容性

推荐使用最新版 Chrome / Edge / Firefox（需支持）：

- `File API`
- `Blob`
- `Web Crypto API`（SHA-256）

## 注意事项

- 批量 ZIP 导出依赖 CDN 的 `JSZip`。当网络不可用或加载失败时，会自动回退到“单个下载列表”模式。
- 如果浏览器拦截下载，请允许当前站点下载或使用“单个逐一下载”。
- 超大文件处理时，受浏览器内存限制影响，建议适当减小分片大小。
